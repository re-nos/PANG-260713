# FEATURE — 메인 화면 (첫 화면 구성)

## 1. 목적

게임 실행 시 최초로 노출되는 메인(타이틀) 화면의 구성을 정의한다. 플레이어가 게임을 시작하기 전 확인할 수 있는 진입점이며, 이후 확장될 메뉴/설정 화면의 기준이 되는 최소 구성을 다룬다.

## 2. 화면 구성 요소

| 영역 | 설명 |
|---|---|
| 타이틀 로고 | 게임 제목("PANG")을 화면 상단 또는 중앙에 강조 표시 |
| 시작(Start) 버튼 | 클릭/입력 시 Mission 1(첫 스테이지)로 진입 |
| 배경 | 게임 분위기를 나타내는 단색 또는 간단한 배경 이미지 (초기 버전은 단순 색상/그라디언트로 대체 가능) |
| 버전/저작권 정보(선택) | 화면 하단에 작게 표기 |

## 3. 인터랙션 흐름

1. 앱 최초 진입 시 메인 화면이 렌더링된다.
2. 사용자가 시작 버튼을 클릭하거나 지정된 키(예: Enter, Space)를 입력하면 Mission 1 게임 화면으로 전환된다.
3. (향후 확장) 게임 오버 또는 클리어 후 "메인으로" 버튼을 통해 다시 메인 화면으로 복귀할 수 있다.

## 4. 초기 구현 범위 (MVP)

- 타이틀 텍스트 + 시작 버튼만으로 구성된 단일 화면.
- 라우팅/상태 전환은 컴포넌트 내부 state(예: `screen: 'main' | 'game'`)로 최소 구현하며, 별도 라우터 라이브러리는 도입하지 않는다.
- 사운드, 애니메이션, 설정 메뉴 등은 이후 미션 확장 단계에서 추가 검토한다.

## 5. 향후 확장 아이디어

- 최고 점수(하이스코어) 표시
- 조작법 안내 화면
- 미션 선택 화면 (Mission 1, 2, 3 ... 중 선택)
