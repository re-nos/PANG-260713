# PLAN — Phase별 개발 목표 (초안)

이 문서는 PANG 게임(Mission 1)을 어떤 순서로 만들어 갈지에 대한 계획입니다. 각 Phase는 **그 자체로 실행 가능한 SW**가 되는 것을 목표로 하며, Phase가 끝날 때마다 고객님께서 직접 실행해 보시고 피드백을 주시는 방식으로 진행합니다.

관련 규칙 문서: [`docs/FEATURES/main.md`](FEATURES/main.md), [`docs/FEATURES/game_rule.md`](FEATURES/game_rule.md), [`docs/FEATURES/mission1.md`](FEATURES/mission1.md)

---

## Phase 1 — 메인 화면 & 화면 전환 골격

**만드는 것**
- 게임 실행 시 보이는 메인(타이틀) 화면 (타이틀 텍스트 + 시작 버튼).
- 시작 버튼을 누르면 "게임 화면"으로 전환되는 최소 골격 (풍선이나 캐릭터는 아직 없어도 됨).

**고객님이 확인해 주실 부분**
- 앱을 처음 켰을 때 타이틀 화면이 정상적으로 보이는지.
- 시작 버튼을 눌렀을 때 화면이 자연스럽게 전환되는지 (에러 없이, 버튼이 눌리는 느낌이 있는지).
- 이 단계에서는 아직 게임 플레이가 없으므로, "화면이 잘 넘어가는지"만 봐주시면 됩니다.

---

## Phase 2 — 캐릭터 이동 & 발사 동작

**만드는 것**
- 게임 화면에 플레이어 캐릭터가 등장하고, 좌우 방향키(또는 지정 키)로 이동 가능.
- 발사 키를 누르면 작살(무기)이 화면 위쪽으로 발사되는 동작 구현 (아직 맞출 대상인 풍선은 없음).

**고객님이 확인해 주실 부분**
- 캐릭터가 좌우로 자연스럽게 움직이는지, 화면 밖으로 나가지 않고 벽에서 잘 멈추는지.
- 발사 키를 눌렀을 때 작살이 위로 똑바로 나가는지, 연타 시 반응이 끊기지 않는지.
- 조작감(반응 속도, 딜레이 유무)이 불편하지 않은지 위주로 봐주시면 됩니다.

---

## Phase 3 — 풍선 등장 & 분열/소멸 로직

**만드는 것**
- 화면에 풍선(적)이 등장하여 중력에 따라 튕기며 움직임 (바운스 물리).
- 작살이 풍선에 맞으면 규칙대로 더 작은 풍선 2개로 분열하거나, 가장 작은 크기면 소멸.

**고객님이 확인해 주실 부분**
- 풍선이 튕기는 움직임이 부자연스럽거나 이상하게(벽 통과 등) 동작하지 않는지.
- 풍선을 맞췄을 때 크기에 맞게 잘 분열되는지, 가장 작은 풍선은 한 번에 사라지는지.
- 이 단계에서는 아직 "닿으면 죽는" 규칙은 없을 수 있습니다 — 풍선을 맞히는 손맛과 물리적 움직임 위주로 확인해 주세요.

---

## Phase 4 — 접촉 판정 & 생명(라이프) 시스템

**만드는 것**
- 캐릭터가 풍선(어떤 크기든)에 닿으면 생명을 1개 잃고 스테이지가 재시작되는 로직.
- 화면에 남은 생명 개수 표시, 생명이 모두 소진되면 게임 오버 처리.

**고객님이 확인해 주실 부분**
- 풍선에 닿았을 때 "닿았다"는 판정이 너무 억울하거나(살짝 스친 정도로 죽음) 너무 관대하지(실제로 부딪혔는데 안 죽음) 않은지.
- 생명이 줄어드는 것이 화면에 잘 보이는지, 스테이지 재시작이 자연스러운지.
- 생명이 0이 되었을 때 게임 오버 처리가 잘 되는지 (메인 화면 복귀 등).

---

## Phase 5 — Mission 1 완성 (클리어 조건 포함 전체 플레이 루프)

**만드는 것**
- Mission 1 난이도 기준(`docs/FEATURES/mission1.md`)에 맞춘 풍선 개수/크기/속도 적용.
- 화면 내 모든 풍선을 제거하면 "클리어" 처리 및 클리어 화면/메시지 표시.
- 메인 화면 → 게임 시작 → 플레이 → 클리어 또는 게임 오버 → 메인 복귀까지 이어지는 전체 흐름 완성.

**고객님이 확인해 주실 부분**
- 시작부터 끝까지 한 번의 플레이 흐름을 처음부터 끝까지 직접 플레이해 보시고, 실제로 "게임처럼" 느껴지는지 전체적인 재미와 난이도를 봐주세요.
- Mission 1이 너무 쉽거나 너무 어렵지 않은지 (초반 스테이지로서 적절한지).
- 클리어했을 때와 게임 오버되었을 때 각각 안내가 명확한지.

---

## 향후 확장 (Phase 5 이후, 범위 외)

아래 항목은 Mission 1 완성 이후 우선순위에 따라 별도 Phase로 진행할 예정이며, 현재 계획에는 포함하지 않습니다.

- 점수 시스템 및 하이스코어 표시
- 파워업 아이템 (Double Wire / Power Wire / Vulcan)
- Mission 2 이후 스테이지 확장

---

## 진행 방식 안내

- 각 Phase가 끝나면 실행 가능한 빌드(또는 `npm run dev`로 확인 가능한 상태)를 공유드립니다.
- 고객님께서 직접 플레이해 보시고 피드백(버그, 조작감, 난이도 등)을 주시면, 다음 Phase 착수 전 또는 병행하여 반영합니다.
- Phase 순서나 범위는 피드백에 따라 조정될 수 있습니다.
