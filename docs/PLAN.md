# PLAN — Phase별 개발 목표 (초안)

이 문서는 PANG 게임(Mission 1)을 어떤 순서로 만들어 갈지에 대한 계획입니다. 각 Phase는 **그 자체로 실행 가능한 SW**가 되는 것을 목표로 하며, Phase가 끝날 때마다 고객님께서 직접 실행해 보시고 피드백을 주시는 방식으로 진행합니다. 이전 계획보다 한 Phase에서 검증할 내용을 더 작게 쪼개어, 매번 짧고 명확하게 확인하실 수 있도록 구성했습니다.

관련 규칙 문서: [`docs/FEATURES/main.md`](FEATURES/main.md), [`docs/FEATURES/game_rule.md`](FEATURES/game_rule.md), [`docs/FEATURES/mission1.md`](FEATURES/mission1.md)

---

## Phase 1 — 메인 화면 UI

**만드는 것**
- 게임 실행 시 보이는 타이틀 화면 (게임 제목 + 시작 버튼)의 화면 구성만 구현.

**고객님이 확인해 주실 부분**
- 앱 실행 시 타이틀 화면이 에러 없이 잘 보이는지.
- 제목/버튼의 배치가 어색하지 않은지 (디자인 완성도보다는 "화면이 뜨는지" 위주로 확인).

---

## Phase 2 — 메인 → 게임 화면 전환

**만드는 것**
- 시작 버튼 클릭 시 (아직 아무 것도 없는) 게임 화면으로 전환되는 로직.

**고객님이 확인해 주실 부분**
- 시작 버튼을 눌렀을 때 화면이 끊김 없이 전환되는지.
- 여러 번 눌러도(연타) 에러 없이 동작하는지.

---

## Phase 3 — 캐릭터 등장 & 좌우 이동

**만드는 것**
- 게임 화면에 플레이어 캐릭터가 나타나고, 방향키로 좌우 이동 가능.

**고객님이 확인해 주실 부분**
- 캐릭터가 자연스러운 속도로 움직이는지.
- 화면 좌우 끝에서 밖으로 나가지 않고 잘 멈추는지.

---

## Phase 4 — 작살 발사 동작

**만드는 것**
- 발사 키 입력 시 캐릭터 위치에서 작살이 위로 발사되어 화면 끝까지 이동 후 사라짐 (아직 맞출 대상 없음).

**고객님이 확인해 주실 부분**
- 발사 키를 눌렀을 때 작살이 똑바로 위로 나가는지.
- 연속으로 눌렀을 때 반응이 끊기거나 밀리지 않는지.

---

## Phase 5 — 풍선 등장 & 바운스 물리

**만드는 것**
- 화면에 풍선이 등장하여 중력/반사에 따라 튕기며 움직임 (충돌 판정은 아직 없음, 시각적 움직임만 확인).

**고객님이 확인해 주실 부분**
- 풍선이 벽/바닥/천장에서 자연스럽게 튕기는지, 화면 밖으로 새거나 멈춰버리지 않는지.
- 움직임 속도와 궤적이 어색하지 않은지.

---

## Phase 6 — 작살-풍선 충돌 & 분열 로직

**만드는 것**
- 작살이 풍선에 닿으면 규칙대로 더 작은 풍선 2개로 분열하는 로직 구현.

**고객님이 확인해 주실 부분**
- 풍선을 맞췄을 때 정확히 갈라지는지 (판정이 너무 빡빡하거나 헐렁하지 않은지).
- 분열된 풍선들이 자연스럽게 반대 방향으로 튕겨 나가는지.

---

## Phase 7 — 최소 크기 소멸 & 스테이지 클리어 감지

**만드는 것**
- 가장 작은 크기의 풍선을 맞히면 소멸 처리.
- 화면 내 모든 풍선이 사라지면 "클리어" 상태를 감지하는 로직 (별도 화면 없이 텍스트/로그 수준도 가능).

**고객님이 확인해 주실 부분**
- 마지막 풍선까지 다 없앴을 때 "클리어" 판정이 정확한 타이밍에 뜨는지.
- 풍선을 다 없애지 않았는데 클리어로 오판하는 경우는 없는지.

---

## Phase 8 — 캐릭터-풍선 접촉 판정 & 스테이지 재시작

**만드는 것**
- 캐릭터가 풍선(크기 무관)에 닿으면 감지하고, 스테이지를 초기 상태로 재시작하는 로직.

**고객님이 확인해 주실 부분**
- 접촉 판정이 실제 부딪힌 느낌과 맞는지 (너무 억울하게 죽거나, 닿았는데 안 죽는 경우가 없는지).
- 재시작 시 캐릭터/풍선 위치가 처음 상태로 잘 초기화되는지.

---

## Phase 9 — 생명(라이프) 시스템 & 게임 오버

**만드는 것**
- 생명 개수 표시 UI, 접촉 시 생명 차감, 생명이 0이 되면 게임 오버 처리.

**고객님이 확인해 주실 부분**
- 생명이 줄어드는 게 화면에 명확히 보이는지.
- 생명이 0이 되었을 때 게임이 멈추고 게임 오버 처리가 되는지 (다음 동작 안내 포함).

---

## Phase 10 — 메인 화면 복귀 흐름

**만드는 것**
- 게임 오버 시 메인 화면으로 돌아가는 처리 (Phase 2에서 만든 전환 로직의 역방향).

**고객님이 확인해 주실 부분**
- 게임 오버 후 메인 화면으로 자연스럽게 복귀되는지.
- 복귀 후 다시 시작 버튼을 눌러 재도전이 정상적으로 되는지.

---

## Phase 11 — Mission 1 난이도 적용

**만드는 것**
- `docs/FEATURES/mission1.md` 기준(풍선 개수/크기/속도 등)에 맞춰 스테이지 값 조정.

**고객님이 확인해 주실 부분**
- 첫 스테이지로서 난이도가 적절한지 (너무 쉽거나 어렵지 않은지).
- 조정된 수치가 실제 플레이에서 의도한 대로 체감되는지.

---

## Phase 12 — Mission 1 전체 플레이 루프 완성

**만드는 것**
- Phase 1~11에서 만든 요소를 하나로 연결: 메인 → 시작 → 플레이 → 클리어/게임 오버 → 메인 복귀까지 끊김 없이 이어지는 완성된 흐름.

**고객님이 확인해 주실 부분**
- 처음부터 끝까지 한 번의 세션으로 직접 플레이해 보시고, 전체적으로 "게임처럼" 느껴지는지.
- 특정 구간에서 튕기거나 멈추는 등 이전 Phase에서 놓친 버그가 없는지 종합 점검.

---

## 향후 확장 (Phase 12 이후, 현재 계획 범위 외)

Mission 1 완성 이후 우선순위에 따라 별도 Phase로 진행 예정이며, 현재 계획에는 포함하지 않습니다.

- 점수 시스템 및 하이스코어 표시
- 파워업 아이템 (Double Wire / Power Wire / Vulcan)
- Mission 2 이후 스테이지 확장

---

## 진행 방식 안내

- 각 Phase가 끝나면 실행 가능한 빌드(또는 `npm run dev`로 확인 가능한 상태)를 공유드립니다.
- 고객님께서 직접 플레이해 보시고 피드백(버그, 조작감, 난이도 등)을 주시면, 다음 Phase 착수 전 또는 병행하여 반영합니다.
- Phase 순서나 범위는 피드백에 따라 조정될 수 있습니다.
